# Day 36 – GitHub Repositories Viewer

## Concepts Learned
- Multiple API calls
- Dependent API calls (user → repos)
- Handling multiple data sets
- Rendering lists dynamically

## Key Learnings
- APIs can depend on each other
- Always clear old UI before rendering new data
- Limit data (slice) for better UI

## Features
- Fetch GitHub user
- Fetch repositories
- Display top repos
- Loading + error handling
